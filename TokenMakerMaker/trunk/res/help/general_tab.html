General Tab

The General tab allows you to enter basic information about your TokenMaker. Here's a breakdown of what each field means:


Field Meaning
Package The package for the generated class. This can be left blank if the class is to be in the default package.
Class Name The name of the class to generate. By convention, this should end in "TokenMaker", for example, "MyLanguageTokenMaker".
Syntax Type The difference between "C-derived syntax" and "All others" is that the former will cause your TokenMaker to auto-indent after lines ending in '{', as well as auto-align closing '}' chars when they are typed (assuming auto-indent is enabled in the RSyntaxTextArea it's running in). It's supposed to be the option you want to pick if the language you are creating derives syntax from C, and uses curly braces to denote code blocks.
Class Comment Used as the Javadoc comment for the generated class. Each line is prefixed with " * " to keep the source looking pretty.
Case-sensitive If checked, the language is considered to be case-sensitive. This means that keywords, functions, etc. are only matched if they are found in the exact same case as they are specified in TokenMakerMaker.
Highlight boolean literals "true" and "false" If checked, then the literals "true" and "false" are colorized according to the RSyntaxTextArea's color scheme as boolean literals. You'll probably want to check this if your language defines these boolean literals.
